# Интеграция с CRM SOTA (U-ON.Travel)

Интеграция отправляет бронирования в CRM, синхронизирует статусы оплаты и подтягивает документы на вылет для раздела «Мои бронирования».

## Настройка

### API-ключ

1. Войдите в [U-ON](https://sota.u-on.ru/) → **Настройки** → **Интеграции** → **API** — активируйте GET/POST.
2. Для прямого U-ON из приложения (только dev): **`EXPO_PUBLIC_UON_API_KEY`** в `.env` / EAS (попадает в `app.config.js` → `extra.uonApiKey`). В production store — только CRM-прокси на сайте, без ключа в бандле.

**Не коммитьте ключ.** В репозитории и в документации реальные значения не указываются.

### Базовый URL

- **Production:** `https://api.u-on.ru` (по умолчанию в коде).
- **Локальные тесты:** задайте `SOTA_CRM_BASE_URL` на URL с реализацией `POST /api/crm/submit-booking` (например локальный `npm run server` из каталога `server/`) или на staging-сайт с CRM-прокси.

После тестов удалите или закомментируйте `SOTA_CRM_BASE_URL`, чтобы снова использовать продакшен API.

## Поток данных

**Источник истины по заявке — CRM (U-ON).** Firestore — кэш после успешного ответа CRM.

1. **Создание бронирования:** `BookingService.createBooking` ставит задачу в **очередь** (`src/services/crm/CrmOutboundQueue.ts`), персистентную в **AsyncStorage**.
2. **Онлайн:** очередь вызывает **request/create** (`sendBookingToCrm`) с **`r_id_internal` = idempotencyKey** (стабильный UUID) — повторные попытки не создают дубликат в CRM.
3. **После успеха CRM** создаётся документ в Firestore (`persistAfterCrmSuccess`: `sotaBookingId`, `idempotencyKey`, `syncVersion`).
4. **Офлайн:** заявка остаётся в очереди; при восстановлении сети обработка возобновляется (`networkService` + `crmOutboundQueue.start` в `App.tsx`).
5. При успешной оплате обновляется статус в CRM (уведомление об оплате, при необходимости — платёж по заявке) — см. `BookingService` и `SotaCrmService`.
6. Документы и список заявок читаются по email/телефону пользователя через API U-ON.

Эндпоинты, поля тел и ограничения — в [**SOTA_CRM_API.md**](./SOTA_CRM_API.md). Официальная документация: [api.u-on.ru/doc](https://api.u-on.ru/doc).

## WebHooks

События CRM (прикрепление файла, уведомления туристу и т.д.) обычно обрабатываются **сервером**, а не приложением. В мобильном клиенте есть заготовки обработчиков в `SotaCrmService`; для приёма webhook настройте URL в кабинете U-ON на ваш бэкенд.

## Ограничения

- До **10 запросов/с** к API.
- Только **HTTPS** (кроме явно заданного mock URL для разработки).
