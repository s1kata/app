# Ручная проверка перед релизом (iOS / Android)

Используйте **production** или **preview** билд на реальном устройстве (не Expo Go), с production URL и секретами EAS.

## Навигация и UI

- [ ] Таб-бар и переходы на экраны бронирования без артефактов.
- [ ] Модальные окна: закрытие системной кнопкой «Назад» на Android; после модалок выбора страны восстанавливается статус-бар.
- [ ] Экраны с полями ввода в модалках: фокус и клавиатура ведут себя ожидаемо (`keyboardShouldPersistTaps`).

## Оплата

- [ ] Диалог перед переходом на внешнюю оплату отображается.
- [ ] После оплаты: возврат по deep link `travelhub://booking-success?...` или закрытие браузера — статус бронирования обновляется после проверки API (не «оплачено» при ошибке сети).
- [ ] Режим «в полёте»: понятное сообщение об ошибке, без краша.

## Нативные возможности

- [ ] Push-уведомления (FCM / APNs настроены для билда).
- [ ] Геолокация: запрос разрешений и поведение при отказе.
- [ ] Длинные списки туров/отелей: прокрутка без сильных лагов.

## Визуал

- [ ] Карточки с тенями и плавающие кнопки на **iOS** и **Android** (отличия `elevation` / `shadow`).

## Конфигурация

- [ ] В билде нет `localhost` в URL оплаты и сайта (`paymentPageUrl`, `websiteBaseUrl`).
- [ ] В production/preview не протекают дев-ключи (`TOURVISOR_TOKEN`, `EXPO_PUBLIC_UON_API_KEY`) и не используются тестовые endpoint-ы.
- [ ] `version` / `versionCode` / `buildNumber` увеличены перед отправкой в стор.

## Store readiness

- [ ] Google Play / App Store карточки заполнены: описание, скриншоты, support URL/email, privacy policy URL.
- [ ] Apple: активна подписка Developer Program, настроены сертификаты/profiles/ASC доступ.
- [ ] Подготовлены релизные заметки и проверено соответствие описания фактическому flow оплаты (внешний веб-платеж, не IAP).
